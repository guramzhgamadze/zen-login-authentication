Passkeys (WebAuthn)

Passwordless sign-in with Face ID, a fingerprint, Windows Hello, or a security key — registered from the Account page. Phishing-resistant and inherently multi-factor, so it skips the password and any 2FA step. Verified locally by the bundled lbuchs/WebAuthn library, no external request. Requires HTTPS.

🔔

New-device login alerts

Emails the account owner the first time their account is signed in from an unrecognised device — like the "new sign-in" alerts from Google or GitHub. Sent with your site's own email, no external service, with a styled, mobile-friendly, customisable body. On by default.

🎨

Redesigned forms

A polished default card, a refined security-blue palette, full-width primary buttons, and clean input focus — and every visual property is still fully controllable from the Elementor toolbar.

Features
Everything you need, nothing you don't
🔒

Complete Auth Flow

Login, registration, lost password, and password reset — all handled on your frontend with proper nonce verification and WordPress hook compatibility.

⚡

Elementor Widgets

Five native Widget_Base widgets — login, registration, lost password, reset password, and account — with full Elementor style controls, V4 compatibility, dynamic content flagging, and Theme Builder integration via real WordPress pages.

🛡

Rate Limiting

Per-form lockout with individual thresholds — set login=5, register=10, lost-password=3, or disable per form. Spoof-resistant client IP detection (real socket address by default; Cloudflare header is opt-in). Optional "count all submissions" mode closes the email-spam loophole on lost-password requests.

🕷

Honeypot Protection

Rotating hidden field (hourly key rotation via HMAC) catches bots. Trapped submissions get a fake success response — bots never know they failed.

📱

Two-Factor Authentication

Opt-in TOTP 2FA per user, set up from the Account page with a locally rendered QR and one-time recovery codes. Enforced across the plugin's forms, AJAX, and wp-login.php; REST/XML-RPC application passwords and Google sign-in are exempt.

🔑

Passkeys (WebAuthn)

Optional passwordless sign-in with Face ID, a fingerprint, Windows Hello, or a security key — registered from the Account page and verified locally with no external request. Phishing-resistant and multi-factor, so it skips the password and any 2FA step. Requires HTTPS.

🔔

New-Device Login Alerts

Email the account owner the first time their account is signed in from an unrecognised device, using your site's own email — no external service. Styled, mobile-friendly, and customisable. On by default.

🤖

Cloudflare Turnstile

Optional privacy-first bot challenge on login, registration, and lost-password — with server-side verification and an encrypted secret key. Works on the plugin's forms and wp-login.php.

🔑

Breached-Password Blocking

Optionally reject leaked passwords via the Have I Been Pwned range API using k-anonymity — only a 5-character hash prefix is sent, never the password. Fails open if the service is down.

🔐

Anti-Enumeration & XML-RPC

Blocks author-scan URLs and the guest REST user listing, makes login errors generic so usernames can't be confirmed, and offers an optional XML-RPC lockdown. On by default where it's safe.

🔌

AJAX Forms

Optional no-reload form submission. Inline error display, loading spinners, and automatic redirect on success — all configurable from the admin panel.

🌐

URL Rewriting

All wp-login.php links across your entire site — login_url, logout_url, lostpassword_url, site_url — are transparently rewritten to your frontend pages.

👥

Multisite Ready

Network activation support. Per-site settings, signup/activation URL rewriting, and automatic option seeding for new sub-sites via wp_initialize_site.

♻

3rd-Party Compatible

Two-factor auth and Turnstile are built in — and the plugin still fires the standard WordPress hooks (login_form, register_form, etc.), so other 2FA, CAPTCHA, and social-login plugins render their fields inside your forms too. MCP Bridge and OAuth flows are exempted from URL rewriting.

♿

Accessible

ARIA roles, aria-required, aria-live regions for errors, focus-visible outlines, and translatable password-strength labels. Clean, semantic HTML throughout.

🔁

Cache-Aware

Automatically excludes auth pages from LiteSpeed Cache, Super Page Cache, WP Rocket, W3 Total Cache, and WP Super Cache. Purges stale 404s on plugin update. Sends no-store HTTP headers as a universal fallback.

📌

Smart Redirects

Runs the standard login_redirect filter on every login, so membership/LMS plugins are respected. Subscribers are kept out of wp-admin (sent to a destination you set), while admins keep their normal flow. Fully honours ?redirect_to=.

👤

Account Management

Logged-in users edit their first/last name, public display name, email, and password from the frontend — a real /account/ page, an Elementor widget, and a classic widget. Guests are bounced to login and returned after signing in.

📊

Login Activity Dashboard

A dashboard widget summarising successful logins, failed attempts, and lockouts over the past week, with top failed usernames and most-blocked IPs. Anonymised IPs, auto-pruned history, and a per-widget on/off panel in Settings.

Quick Start
Up and running in 60 seconds
# Upload and activate
wp plugin install zen-login-authentication.zip --activate

# Rewrite rules flush automatically on first load (v1.4.16+)
# Run manually if needed:
wp rewrite flush

# Auth pages are created automatically on activation
# (existing pages with matching slugs are adopted, not duplicated)

# Optional: enable AJAX forms and user-chosen passwords
wp option update zenlogau_use_ajax 1
wp option update zenlogau_user_passwords 1

# Optional: set the subscriber post-login destination
# (Settings → Zen Login & Authentication — slug or URL — or via filter:)
add_filter( 'zenlogau_subscriber_redirect', fn() => home_url( '/dashboard/' ) );

# Optional security: breached-password checks + XML-RPC lockdown
# (two-factor auth is on by default; each user enrols on the Account page)
wp option update zenlogau_block_breached 1
wp option update zenlogau_disable_xmlrpc 1

# That's it. Visit /login/ on your site.
